# i18n

## Locales

| Code | Name | Status |
| --- | --- | --- |
| `ru` | Russian | **default**, authoritative copy |
| `en` | English | maintained alongside ru |
| `uk` | Ukrainian | maintained alongside ru |

## Library

- `next-intl` on the web side.
- Catalogs: `apps/web/messages/{ru,en,uk}.json`.
- Request config: `apps/web/src/i18n/request.ts`.
- Middleware: `apps/web/src/middleware.ts` — locale prefix is always present
  (`/ru`, `/en`, `/uk`); the root `/` redirects to `/ru`. If `NEXT_LOCALE`
  stores a supported locale and a stale prefixed URL is opened with a
  different locale, middleware redirects to the saved locale prefix.

## Authoring rules

1. Add the key to **all three** files in the same change. Untranslated
   values may use `[TODO] …` as a placeholder, but the key must exist.
2. Namespace by feature: `training.session.start`, `drill.library.title`.
3. Use ICU MessageFormat for pluralization and gender.
4. Dates/numbers — use `useFormatter` (web) or pass server-formatted
   strings from the API. Do not call `toLocaleString` directly.
5. Renames/removals must update all three files together.

## Server vs client

- Server components: `getTranslations`, `setRequestLocale`.
- Client components: `useTranslations`.
- Always pass `params.locale` through; do not infer from headers in components.

## API responses

The API should return data-only payloads. User-visible text comes from the
web catalog. When the API needs to surface a message (validation, system
errors), it returns a stable `code` string; the web translates it.

Preset drill templates are also localized on the web side. The API returns the
stored template data; `apps/web/src/lib/drill-localization.ts` maps known system
drills by `cmseeddrill...` id and demo preset drills by stable layout/name keys
to `systemDrills` entries in all locale catalogs. User-created drill templates
remain literal user content and are not translated automatically.

Example:
```json
{ "error": { "code": "training.session.alreadyFinished" } }
```

The web catalog has an `errors.api.training.session.alreadyFinished` key.
