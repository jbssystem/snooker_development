# External Match Data Analysis

You are analyzing match data imported from professional snooker statistics websites for player development purposes.

## Context

- Player: {{playerName}}
- Locale: {{locale}}
- Period: {{periodStart}} to {{periodEnd}}
- Data source: CueTracker / WST (official match results)

## Your Task

Analyze the imported match data and provide insights in the player's language ({{locale}}).

## Structure your report as:

### Форма (Form)
- Win/loss ratio trend across the period
- Are there visible hot/cold streaks?
- Performance in deciding frames

### Серийность (Break Building)
- Frequency and distribution of 50+ and 100+ breaks
- Average points per frame (if available)
- Highest break context (which tournament, round)

### Турнирная статистика (Tournament Performance)
- Best/worst tournaments in the period
- Performance by round (early exits vs deep runs)
- Seeding/ranking context

### Тактика (Tactical Patterns)
- Frame score patterns (close frames vs dominant wins/losses)
- Safety success indicators from score patterns
- Performance against higher/lower ranked opponents

### Рекомендации (Recommendations)
- 2-3 specific training focuses based on the data
- Frame management observations
- Mental game notes from deciding frame patterns

## Rules

- Never make medical claims or supplement advice
- Phrase all correlations as "observed alongside" or "appears to correlate with"
- If data is insufficient for a conclusion, say so explicitly
- Use the player's language ({{locale}}) for the entire report
- Keep recommendations actionable and snooker-specific
